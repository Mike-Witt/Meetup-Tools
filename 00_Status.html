General Scheme

  • The reason for the separate "member dictionary" and "location dictionary" is so that we can (in the member dict) maintain the original form of the information obtained from meetup, and try different techniques to parse it, without having to hit the meetup sites again.

  • Make-Member-Dictionary.py group-name
    - Create dictionary with member info and "raw" meetup location string
    - This is the only code that needs to "scrape" meetup

  • Make-Location-Dictionary.py group-name
    - Read a member dictionary and write a location dictionary
    - Group members into locations

  • Find-Locations.py group-name
    - Does misc processing (eg: "unquote" names with "%" codes)
    - Add geographic coordinates to a location dictionary
    - This is the only code that needs to access geo databases

  • Map-Members.py -flag group-name
    - Read a location dictionary and write a .kml file
    - This also does the sorting, merges duplicate locations, and fixes "%" codes in city names (not in that order).
